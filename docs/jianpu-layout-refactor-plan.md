# 重构方案:简谱排版(和弦/八度点/时值修饰 不重叠)

> 状态:方案已定,自主实施。存档供后续参考。
> 关联问题:简谱区域排版重叠(和弦、高音八度点、下划线/短横/tie)。

## 现状问题(puppeteer 坐标量化确认)

1. **八度点与数字重叠**:高音区和弦时,上方声部八度点(yRow-15-d*7)与该声部数字
   及相邻声部数字边界重叠 4-5px。低音点(yRow+10+d*7)同理。
2. **时值修饰不随声部**:下划线/短横/附点整组画一次,固定在 baseY。和弦纵排时,
   下划线只画在中间(baseY),与上下声部数字关系混乱(应每个声部各自画,或整组
   画在最低声部下方)。
3. **tie 固定 baseY-13**:和弦多声部 tie 会挤在同一 y。
4. **tuplet 标记固定 baseY-28**:和弦纵排较高时与八度点重叠。
5. **临时记号 accOffset 随 slot 变**:窄 slot 时侵入相邻音符。

## 重构方向

核心思路:**简谱组的垂直布局要「盒子化」**——计算整组(含所有声部 + 各自八度点 +
临时记号)的总高度,居中放置,内部元素按声部相对偏移,时值修饰画在整组盒子的
统一位置(下方)。

### 具体改动(jianpu.ts renderJianpuSVG)

1. **声部行高动态化**:CHORD_LINE 当前固定。改成根据声部是否有八度点动态留白:
   有高八度的声部上方多留 1 行(给点),有低八度的下方多留。
2. **八度点位置随声部 yRow,但确保不侵入相邻声部**:八度点 y 基于该声部 yRow,
   点间距 7px。若纵排行高足够(含八度点空间),不会重叠。
3. **时值修饰(下划线/短横/附点)画在整组「最低声部」下方**:不再固定 baseY,
   而是取纵排后最低声部的 yRow,在其下方画。这样下划线与所有数字都不挤。
   - 附点:画在首声部(最高音)右侧,或整组中心右侧。
4. **tie 弧线**:每个声部独立画弧(按声部 yRow),而非固定 baseY-13。
5. **tuplet 标记**:画在整组「最高声部 + 八度点」之上,动态避让。
6. **临时记号 accOffset**:固定最小值,不随 slot 缩到侵入邻居。

### 盒子模型(伪码)
```
对每个时间位 [s0,s1]:
  members = 组内声部(含简谱信息)
  按 midi 排序(高音在上)
  算每个声部的「占高」= 数字高 + (上方八度点? 点高:0) + (下方八度点? 点高:0)
  组总高 = sum(各声部占高) + 声部间间隙
  组顶部 y = baseY - 组总高/2(居中)
  逐声部画:数字在声部行 y,八度点在 y±偏移(声部占高内)
  时值修饰:画在最低声部 y 下方
```

### 验证(不靠视觉)
- puppeteer 抓 jp-elem bbox,断言:
  - 同声部的八度点与数字不重叠(点在数字外)
  - 相邻声部数字间有间隙(行高足够)
  - 下划线在最低声部数字下方
  - tuplet 标记在最高元素上方
- beam-test 用例 49/50/51/53 重叠数下降(八度点重叠归零)
