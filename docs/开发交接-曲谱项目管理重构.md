## 二、MeasureSelector 组件规格(核心,未完成)

组件文件:`src/ui/measure-selector.ts`,挂在 demo 的 `src/library-demo.ts`。
API:`buildMeasureSelector(state: {totalMeasures, start, count, hasContent}, callbacks: {onChange, onDeleteMeasure, onAddMeasure}) → {el, refresh, setSelection}`。

### 2.1 组件要实现的全部交互(逐个标完成度)

**① 拖左右把手(改 start/count)** — ✅ 已实现,但有 bug
- 拖左把手:改 start(选区左边界),吸附到小节边界。收缩到 count=0 = 删末尾小节。
- 拖右把手:改 count(选区宽度=每行小节数),吸附到小节边界。收缩到 0 = 删末尾小节。
- 实现:`startDrag('l'/'r')` + `onMove` 逐帧 `apply(false)`。
- **bug**:见 2.2。

**② 拖选框框体横移(平移选区改 start)** — ❌ **完全没实现(缺失!)**
- **预期行为**:pointerdown 在选框框体中部(非把手区域)→ 拖动 → 整个选框横移,start 跟手吸附到鼠标经过的小节边界,count 不变。
- 类似"滑块在书签轴上滑动":选框从当前位置平滑滑到新位置。
- **拖拽阈值区分点击 vs 拖拽**:pointerdown 后移动超 4px = 拖拽,否则抬起 = (框体内点击无意义,可忽略或也当作跳转)。
- **动画规格**(见 2.3 拖拽框体)。
- **实现要点**:在 `sel`(选框元素)上加 pointerdown 监听(排除把手区域),onMove 时按鼠标 x 算目标 start(吸附),`apply` 重算。

**③ 点书签跳转(选框起点跳到该书签)** — ✅ 已实现
- pointerdown 在书签上,移动 < 4px 抬起 = 点击 → 选框 start 跳到该书签(count 不变,clamp 到 maxStart)。
- 动画:选框滑动到新书签位置(~250ms 弹性)。
- 实现:`downInfo` + `onUp`。

**④ 加小节(末尾 +)** — ✅ 已实现
- 点末尾 `+` 按钮 → totalMeasures+1,新书签从右侧出现。
- **动画(未实现)**:新书签应 alpha+半缩放进场(0.7→1, opacity 0→1,~200ms)。当前只靠 `apply(true)` 的 transform 过渡,没有专门的进场动画。

**⑤ 删小节(框外书签 hover 出 ×)** — ✅ 基本实现,但有 bug
- 框外书签 hover 显示红色 ×(弹性缩放显隐),框内书签无 ×。点 × 删该小节,后续小节前移。
- **动画(未实现)**:被删书签应 alpha+缩放退场,后续书签滑动补位。当前只靠 transform 过渡。
- **bug**:见 2.2(选框宽度/位置算错)。

### 2.2 已知 bug(逐个,含根因 + 修法)

**Bug A:横滑滚动不工作(确认真 bug)**
- 现象:多小节时选框区无法横滑;`scrollLeft` 设 200 读回 0;`scrollWidth=clientWidth`。
- 根因:组件所有子元素(书签/选框/+)都 `position:absolute`,**absolute 子元素不撑开父元素 scrollWidth** → wrap 无可滚动内容。加了 `.ms-track`(流式,设 width)想撑开,但 `trackWidth=clientWidth`(没生效)。
- **修法**:确保 `.ms-track` 的 `style.width=totalW` 真生效(检查 `min-width:100%` 是否覆盖、totalW 是否算对);或改用非 absolute 内容撑开。**加小节后滚动跳回 0 同根因**。

**Bug B:拖选框框体横移 — 缺失(见 ②,根本没实现)**

**Bug C:删小节后选框宽度/位置算错**
- 现象:删小节后 totalMeasures 变了,选框的 start/count 可能越界 → `computeX` 算出错误的 selX/selW → 选框宽度/位置不对。
- 书签 transform 重排本身可能 OK(puppeteer 下连续、补位正确),但**选框的 selX/selW 没正确 clamp/重算**。
- **修法**:删除后 `state.start` clamp 到 `max(0, total-count)`、`state.count` clamp 到 `min(count, total-start)`,再 `apply`;验证选框始终正确包住当前选中书签(selX ≤ 第一个选中书签左缘,selRight ≥ 最后一个选中书签右缘)。

**Bug D(可能):拖把手时 hover 删除叉遮挡**
- puppeteer 下把手和删除叉不重叠,但真机需确认:hover 删除叉显形(opacity:1)后,它是否在某些位置挡住把手 pointerdown。**用真实浏览器确认**。

### 2.3 动画规格(用户确认,逐场景)— 部分未实现,需完善

| 场景 | 预期动画 | 完成度 |
|---|---|---|
| **拖把手(改 start/count)** | 选框边界(把手)**完全跟手**(逐帧 `apply`,transition:none)。**书签跨格(框外↔框内)带短动画(~100ms 弹性)**平滑滑入/滑出。**跨框距离突变**(书签被选框推开)带动画(~200ms)。抬手后选框恢复 transition。 | ⚠️ 跟手做了;书签跨格/跨框动画的**实际观感未验证**(transition 写了但没确认效果) |
| **拖框体横移(改 start)** | 选框**滑动**到新位置(~250ms 弹性),书签跟随重排(跨格动画)。 | ❌ 完全没做(框体横移缺失) |
| **点书签跳转** | 选框**滑动**到新书签(~250ms 弹性),书签本身不动。 | ✅ 做了(`apply(true)`) |

**未实现动画的修法**:
- 加/删小节的 alpha+缩放:新书签创建时初始 `opacity:0; transform: translateX(x) scale(0.7)`,rAF 后过渡到 `opacity:1; scale(1)`;删除时先 `opacity:0; scale(0.7)` 等过渡完再移除 DOM。
- 拖框体横移:见 ②。
- 拖把手的书签跨格动画:确认 transition 在 DOM 移动(inside↔outside)时是否生效(appendChild 可能打断 transition,需用 FLIP 或 transform 而非 DOM 移动)。
