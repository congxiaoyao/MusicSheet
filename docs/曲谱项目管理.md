# 曲谱项目管理(整曲存储 + 跨设备共享)

本功能支持完整的曲谱项目(整曲多小节)编辑,并通过局域网服务器实现跨设备共享:
笔记本上编辑,pad/电视上查看同一份数据。

## 架构概览

```
笔记本(编辑)                         pad/电视(查看)
   │                                      │
   │  浏览器 http://localhost:5173(dev)    │  浏览器 http://<笔记本IP>:4173(生产)
   │       │                              │       │
   │       │ /api 代理                    │       │ /api(同进程)
   │       ▼                              │       ▼
   │   Vite dev server ──proxy──┐         │   Node 服务器
   │                            ▼         │   (server/index.mjs)
   │                      Node 存储服务器  │      │
   │                      (4173, 0.0.0.0) │      ▼
   │                             │        │   store/pieces/<id>/
   │                             ▼        │     ├─ manifest.json
   │                      store/pieces/ ◀─┤     ├─ m0001.json
   │                      (每小节一文件)  │     └─ m0002.json ...
```

- **Node 存储服务器**(`server/index.mjs`):纯 Node `http`,无框架。绑 `0.0.0.0:4173`。
  开发时经 Vite proxy(`/api` → `localhost:4173`);生产时同进程既服务 API 又托管 `dist/`。
- **存储格式**:每曲谱=一个目录 `store/pieces/<id>/`,`manifest.json` + 每个小节一个 `m{NNNN}.json`。
  局部更新=覆写单个小节文件(硬指标:不重写整曲)。

## 启动

### 开发(本机)

```bash
npm run dev     # 同时起 Node 服务器(4173)+ Vite(5173)
```

> 若 5173 被占用,Vite 会顺延到 5174/5175...。Vite 把 `/api` 代理到 4173,前端无感知。

### 生产(跨设备访问)

```bash
npm run build   # 打包前端到 dist/
npm start       # Node 服务器托管 dist/ + 提供 /api,绑 0.0.0.0:4173
```

启动后控制台会打印局域网访问地址,例如:
```
  本机访问:    http://localhost:4173
  局域网访问:  http://192.168.31.227:4173   ← pad/电视 用这个
```

在同一局域网的 pad/电视浏览器打开该地址即可查看/编辑同一份曲谱数据。

## 数据模型

整曲(`src/core/score.ts`):
```ts
interface MeasureData { treble: Note[]; bass: Note[]; }   // 单个小节
interface ScoreMeta { id, title, key, time, totalMeasures, viewMode, updatedAt }
interface Score { meta: ScoreMeta; measures: MeasureData[]; }  // measures 长度=totalMeasures
```

**编辑区范围**(`rangeToPiece`):整曲存 N 个小节,编辑时只把「第 startMeasure 起的连续
`measuresPerLine` 个小节」拼成一个临时 `Piece`,喂给现有渲染/编辑/播放层(零改动)。
点小节书签/预览小节 → 换 startMeasure → 编辑区换到「从该小节起的 N 个连续小节」,其它小节保留。

编辑后(`pieceBackToScore`)按小节拍边界切回 `score.measures`,3 秒防抖后按小节局部落盘。

## API 端点

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/pieces` | 列所有曲谱 meta |
| POST | `/api/pieces` | 建新曲谱 |
| GET | `/api/pieces/:id` | 读整曲 Score |
| PUT | `/api/pieces/:id/meta` | 改 meta(改 time/totalMeasures 时重建/截断小节) |
| PUT | `/api/pieces/:id/measures/:n` | **单小节局部覆写**(硬指标) |
| DELETE | `/api/pieces/:id` | 删整曲 |
| GET | `/api/pieces/:id/export` | 导出整曲(.mscore 文本) |
| POST | `/api/pieces/:id/import` | 整曲打包导入(覆盖该曲谱) |
| GET | `/api/lan-ip` | 本机局域网 IP 列表 |

## 验证脚本

每个核心功能都有 puppeteer 验证脚本(`scripts/verify-*.mjs`),需先起 `npm run dev`:

| 脚本 | 验证 |
|---|---|
| `verify-score.mjs` | 数据模型:rangeToPiece/pieceBackToScore 往返(含三连音浮点鲁棒) |
| `verify-persist.mjs` | 编辑 → 3 秒防抖 → 按小节局部落盘 → 刷新恢复 |
| `verify-measure-nav.mjs` | 小节书签导航:点第3小节编辑 → 音落进 m0003(其它不动) |
| `verify-preview-modal.mjs` | 整曲预览弹窗:多行渲染 + 五线谱/简谱切换 + 点小节跳转 |
| `verify-io.mjs` | 曲谱级导入/导出往返 + 删除 |
