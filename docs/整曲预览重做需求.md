# 整曲预览重做需求(纸质谱目标)

## 目标
把整曲预览从"堆单行渲染器"重做成**像一张纸质乐谱**(engraved score)。
独立演化,只复用 `layout.ts` 的核心几何常量(SS/FONT/prefix 几何)和 `positionInBar` 拍位计算,**不侵入** `computeLayout`/`staff.ts`。

## 现有问题(必须解决)
1. 每行都画完整谱号+调号+拍号 —— 真谱只在第一行有完整前缀,后续行只画谱号(调号按惯例,拍号通常省略)。
2. 每行末尾都是终止线(粗+细) —— 只有曲末才是终止线,中间行末是普通小节线。
3. 小节宽度机械等分(每行固定 N 个,等宽) —— 真谱小节宽度按内容自适应,音符密的更宽。
4. 没有小节号、没有 grand staff 的连谱号 `{`、没有系统间留白。
5. 点击定位靠 contentLeft/contentRight 等分换算,粗糙,不是命中真实小节几何。

## 纸质谱特征(要实现的)
1. **system(系统)概念**:每行是一个 system,系统间距 > 小节内间距。
2. **前缀分级**:首 system 全前缀;后续 system 仅谱号(+调号)。需给 layout 传 `prefixMode: 'full' | 'clef-only'`。
3. **终止线受控**:仅曲末画终止线,中间行末普通小节线。需给渲染层 `finalBarline` 标志。
4. **小节号**:每隔 4/8 小节标号。
5. **grand staff 连谱号**:双谱表左侧的 `{` + 粗竖线。
6. **系统间距**:系统间留白,像纸质谱。
7. **页面化**:真正边距(可选 P3 真分页)。

## 实现路径(独立演化)
- 新增 `computeLineLayout(score, lineChunk, width, prefixMode)`:复用 layout.ts 常量 + positionInBar,只布局该行的小节。
- 新增 `renderLineSystem(...)`:独立渲染一行 system(staff + jianpu + clef/key/time/barline),终止线由 `finalBarline` 参数控制。
- 新增 `buildFullScoreSVG(score, {mode})`:每行 renderLineSystem → `<g transform="translate(0, 累积高)">` 垂直堆叠 → 单个 `<svg>`。
- 每小节组挂 `data-measure="n"` 供点击定位(命中真实小节几何,非等分换算)。
- treble/bass 双谱表:每行内 treble/bass 上下堆叠 + 左侧连谱号。

## 分阶段
- **P1**:终止线只曲末 + 后续行只画谱号 + grand staff 连谱号 + 系统间距。改完预览立刻"像谱子"。
- **P2**:小节号 + 点击命中真实几何。
- **P3**:小节宽度自适应 + 真分页(为未来图片导出铺路)。

## 复用边界(独立演化,不侵入现有)
- **复用**:layout.ts 的 FONT/SS/PAD/prefix 几何常量、positionInBar、slotWidthFor、resolvePitch;core 的 noteStartBeats/beatsPerBar。
- **不侵入**:computeLayout、staff.ts、jianpu.ts 的现有签名不动(单行编辑保持零回归)。
- 允许在新文件 full-score.ts 内重写部分渲染原语(画五线、符头),换取对纸质谱特征的完全控制。

## 优先级
低于 MeasureSelector 组件。组件打磨完成后做。
