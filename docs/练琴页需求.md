# 练琴页需求

## 一、定位

练琴页是编辑完成后的**主要练琴场景**，独立页面，从编辑器进入。

- **层级**：曲谱库 → 编辑器 → 练琴（第三级页面）
- **角色**：编辑器负责"把曲子做出来"，练琴页负责"把曲子练熟"
- **只读**：练琴页不改谱，要改回编辑器
- **横屏优先**：面向笔记本 / 平板横放 / 电视

---

## 二、核心理念

### 2.1 为什么需要练琴页

读谱能力不足的初学者，看到五线谱上的音符，大脑要走一条慢且易断的翻译链路：

> 第几线第几间 → 是什么音 → 哪个八度 → 该按哪个键 → 用哪个手指

这条链路一旦卡住，练琴就变成挫败。

### 2.2 练琴页怎么解决

用**键位提示短路这条链路**——不让用户去"读"谱，而是在正确时机直接用视觉告诉他"现在按这个键"。同时保留谱面，让用户在跟弹中**被动吸收**音与键位的对应关系，逐步建立读谱能力。

### 2.3 主视图：瀑布流 + 谱面融合（已确认）

主视图是音游式键位提示与纸质谱面的融合体，两者不是并列，而是**叠加**：

- **谱面**占据屏幕主体，多行纸质谱从上到下铺开
- **当前演奏行**清晰，其余行半透明（卡拉 OK 式聚焦）
- **瀑布流方块**从当前行底部开始，向下掉落到键盘上方的判定线
- **键盘**在屏幕底部，方块落点对应的键被高亮

三者构成一条连贯的视觉链：谱面（认音）→ 方块（按键提示）→ 键盘（按键位置）。

## 三、页面结构

### 3.1 布局

```
┌──────────────────────────────────────────────────┐
│ ← 曲名  调号拍号 BPM    播放/进度/变速/模式   ⚙   │ 顶部控制栏
├──────────────────────────────────────────────────┤
│                                                  │
│  ▍当前行（清晰）—— 谱号 调号 小节 音符            │
│                                                  │
│   （半透明）后续行向下铺开                        │ 谱面区（主体）
│                                                  │
│        ▓▓      ▓▓         方块向下掉落            │
│   ━━━━━━━━━━━━━━━ 判定线                        │ 瀑布流区
│   🎹🎹🎹🎹🎹🎹🎹  键盘（高亮当前音）              │
└──────────────────────────────────────────────────┘
```

- **顶部控制栏**：分主行 + 副行，承载核心控制。底部不放控制条，把空间全留给谱面和键盘。
- **谱面区**：屏幕主体，铺满。无卡片容器，谱子直接铺在背景上。
- **瀑布流区**：键盘正上方，方块只在此区域可见，区域外裁掉。
- **键盘**：屏幕底部，占较小高度。

### 3.2 顶部控制栏

控制栏分**主行**和**副行**，层次分明，不拥挤。

**主行**（常驻，高频操作）：

```
┌──────────────────────────────────────────────┐
│ ← 小星星  ♯0 4/4   ⏸           ♩=100    ⚙   │
└──────────────────────────────────────────────┘
   左：返回        中：播放/暂停      右：BPM 设置
       曲名            主按钮              入口
       调号拍号
```

| 位置 | 元素 | 说明 |
|---|---|---|
| 左侧 | 返回、曲名、调号拍号徽章 | 曲目信息，静态为主 |
| 中部 | 播放 / 暂停（单主按钮） | 练琴最高频操作；停止靠快捷键，不占位 |
| 右侧 | BPM 数字、设置入口(⚙) | BPM 仅显示数值，调整在设置面板内 |

**副行**（练习模式开关，独立一行，宽松）：

```
┌──────────────────────────────────────────────┐
│        AB 循环    单手    节拍器              │
└──────────────────────────────────────────────┘
```

练习时频繁切换，故常驻、独立一行，三个开关一字排开，互不挤压。

### 3.3 进度控制：融进谱面

练琴场景下进度条作用被削弱（多数时间用 AB 循环磨某段），不做成独立控件。

- **进度指示 = 当前小节高亮框**（谱面上正在演奏的小节高亮）
- **跳转 = 点击谱面某小节**，播放从该小节继续
- **AB 区间标记**：在谱面对应小节位置标注 A / B 点（而非进度条上的滑块）

### 3.4 设置面板(⚙)

收纳"配一次不常动"的选项，点设置入口展开：

- 变速滑块（降速 / 加速，核心练琴控制，但调整频率低于播放）
- 键盘音域调整（默认按谱面自动，可手动改范围）
- 键面标注方式（音名 / 唱名 / 关闭）
- 视图切换（瀑布流+谱面 / 纯谱面——后者 MVP 预留不实现）
- 其他播放选项（如停止、重置等次要操作）

## 四、谱面行为（已确认）

### 4.1 行滚动（提词器式）

- 谱面按"行"（system）组织，随播放自动滚动
- **当前演奏行始终锁定在屏幕顶部固定位置**（清晰带），不被顶部控制栏遮挡
- 换行时整张谱面平滑上推，像全屏歌词
- 当前行之上留出谱号完整显示的空间；当前行之下铺开后续行

### 4.2 卡拉 OK 式渐变

- **当前行完全清晰**
- 往下**快速过渡**到半透明（约一行高度内完成过渡）
- 半透明行铺满到瀑布流区，作为背景
- 当前行的清晰带位置必须和瀑布流方块区的上边缘对齐——方块从清晰带底部开始掉落，不与当前行重叠

### 4.3 键盘音域

- **默认按谱面音域自动确定**键盘范围（曲子用到哪些音就显示哪些键）
- 支持用户手动调整范围
- 键盘标注：白键标注 C 音名，便于定位

## 五、瀑布流方块（已确认）

### 5.1 横轴 = 键盘位置

- 方块的水平位置就是它该按的键在键盘上的位置
- "看方块 → 按键"是零翻译的，方块落到 C4 上方就是按 C4
- 左右手在物理键盘上天然空间分离（左手靠左、右手靠右），方块自动分轨，不需要额外分轨逻辑

### 5.2 颜色区分左右手

- 右手方块和左手方块用不同颜色区分（如蓝 / 橙）
- 颜色也同步到键盘高亮：右手的音点亮键盘时用右手色，左手同理

### 5.3 方块标注

- 方块上标注音名（初学者不认音，标注降低认知负担）
- *待定：是否只在接近判定线时显示音名，远处不显示（避免干扰）*

### 5.4 掉落与判定

- 方块从当前行底部出现，匀速向下掉落
- **判定线 = 键盘上沿**：方块底边贴到键盘上沿时，就是该按的时刻
- 方块落点对应的键被点亮（高亮）
- 方块区的上边缘做渐变淡出（fade），方块接近上边缘时自然消失，不生硬

### 5.5 方块区范围（关键约束）

- 方块区从**当前行底部**开始，一直到键盘顶部
- 方块区高度 = 当前演奏位置到键盘的距离，**由行高和布局动态计算**，不写死
- 方块区上边缘 = 当前行底部（清晰带底）——方块最高只能到这里，保证不遮挡当前行
- 方块超出区域的部分被裁掉

## 六、播放与练习模式

练琴页的播放基于现有的播放引擎，核心是**给用户掌控感**：L1 阶段没有输入反馈，用户靠控制播放节奏来练习。

### 6.1 基础播放控制

- 播放 / 暂停 / 停止
- 进度条：可拖拽 seek，带小节刻度
- 变速：降速到 0.4x ~ 加速到 1.6x（降速是练琴核心需求）

### 6.2 AB 段循环（练习模式）

- 在进度条上标记 A 点和 B 点，两点之间反复播放
- A / B 点可拖拽调整
- 磨一个难点小节反复练

### 6.3 单手隔离（练习模式）

- 可选择只看 / 只听右手、左手、或双手
- 隔离一只手时，另一只手的方块隐藏、键盘高亮隐藏、播放也可静音该手
- 分手练习

### 6.4 节拍器（练习模式）

- 视觉打拍：判定线随节拍脉冲闪烁
- （可选）打点音
- 可开关

### 6.5 L1 阶段的本质

L1 没有输入源，瀑布流是**自动滚动**的——它不等用户，按 BPM 自己往前。用户弹错了、弹慢了，它照样走。

用户靠**变速（降速练）、AB 循环（磨一段）、暂停**拿回节奏控制权。这三个模式就是"在 L1 约束下给用户掌控感"的手段。

## 七、反馈闭环（分阶段）

练琴的"陪练"能力取决于能否感知用户弹了什么。按输入来源分三个层级：

### 7.1 L1 —— 纯示谱（MVP）

- **不监听任何输入**，纯单向输出：播放 + 键位提示 + 键盘高亮
- 本质是"带键位提示的回放器"，用户自己跟着弹
- 门槛最低，零硬件，MVP 首先实现

### 7.2 L2 —— 麦克风识音（演进）

- 用设备麦克风识别用户弹的音
- 判断对错，给反馈
- 零硬件，但音高识别有精度和和弦识别的难度

### 7.3 L3 —— MIDI 电钢琴（演进）

- 外接 MIDI 键盘，逐音精确判断
- 跟弹评分
- 精度最高，但要求用户有电钢琴
- 项目已有的"固定 C 调指法"设计（配合电钢琴移调）是这一层的伏笔

### 7.4 关键约束：输入源抽象

三个层级本质是**输入来源的不同形式**。练琴内核不关心音从哪来，只订阅一个统一的输入源：

- L1 的输入源是"空"（不订阅，纯示谱）
- L2 换成麦克风识别
- L3 换成 MIDI

判断 / 评分 / 跟弹的逻辑是同一套，只是 L1 不打开它。**MVP 实现时必须预留这个抽象**，让 L2 / L3 接入时内核零改动。

## 九、关键约束汇总

1. **谱面是主角**，不是附属。谱子直接铺满，无卡片容器。
2. **当前行锁定在顶部**清晰带，换行时平滑上推，最后几行也能滚到位（内容不够时谱面整体上移到底）。
3. **卡拉 OK 式渐变**：当前行清晰，往下快速过渡到半透明，过渡在一行高度内完成。
4. **方块横轴 = 键盘位置**，零翻译，左右手天然分轨。
5. **方块区从当前行底部到键盘顶**，高度动态计算不写死，超出裁掉，上边缘 fade。
6. **判定线 = 键盘上沿**，方块落点同步高亮对应键。
7. **键盘音域默认按谱面自动**，可手动调。
8. **控制栏分主行+副行**：主行放曲目信息和播放主按钮，副行放练习模式开关。进度融进谱面，变速/键盘/视图等收进设置面板。
9. **L1 纯示谱，输入源抽象必须预留**，L2/L3 演进时内核不动。
10. **只读**，练琴页不改谱。

## 十、不在本需求范围内

- 进度记录与激励（练琴时长、遍数、通关 BPM 等）——后续单独需求
- 曲谱库卡片的练琴入口展示——后续视情况加
- 纯谱面分页模式——MVP 不实现，仅预留切换口
- L2 麦克风识音 / L3 MIDI 跟弹的产品级细节——演进阶段单独需求

## 附：待定项

- 方块音名标注的显示策略（始终显示 / 仅接近判定线时显示）
- 谱面是否显示简谱档（五线谱 / 简谱 / 双谱），与编辑器预览一致
- 半透明的具体不透明度、渐变过渡的精确范围（需实际观感调校）
