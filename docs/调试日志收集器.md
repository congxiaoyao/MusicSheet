# 调试日志收集器

MeasureSelector 组件内置一套运行时日志收集器,用于排查拖拽/mask/hover 等时序相关问题。
组件挂载时自动初始化,挂在 `window` 上,默认不记录(无业务调用点),调试时手动注入。

## 前置:启动日志接收服务

日志落盘需要 `server/log-sink.mjs`(独立端口 4174,不干扰主 server 4173):

```bash
node server/log-sink.mjs
# [log-sink] 监听 http://localhost:4174
```

服务收到日志后写入项目根 `ms-log.json`(覆盖)。CORS 已开,支持 IP 访问(非仅 localhost)。

## 控制台用法

在浏览器 DevTools Console:

```js
__msLogClear()   // 清空已收集的日志
// ... 操作页面复现问题 ...
__msLogSave()    // 上传到 log-sink,落盘 ms-log.json
__msLog          // 查看当前日志数组(不上传)
__msLogFn        // 拿到 log 函数,可在 console 临时调用记日志
```

## 在代码中记录日志

业务代码默认**不调用** log(生产环境零开销)。调试时在关键点临时加 `log(...)` 调用:

```ts
// src/ui/measure-selector.ts 内部(闭包内 log 可用)
const setBlkHover = (b, hovered) => {
  if (drag) { log('hover_SKIP', { idx: b.idx, hovered }); return; }
  // ...
  log('hover_CLEAR', { idx: b.idx });
};
```

调试完**务必删除 log(...) 调用**(保留收集器本身和 log 函数声明,勿删)。
检查:`grep "log('" src/ui/measure-selector.ts` 应只剩收集器定义,无业务调用。

## 日志格式

每条:`{ t: number(性能时间戳ms), tag: string, data: unknown }`

常用 tag(历史调试用过):
- `drag_start` / `drag_end` — 拖拽起止(mode/start/count)
- `hover_CLEAR` / `hover_RESTORE` / `hover_SKIP` — mask 清除/恢复/被拖拽拦截
- `mask_SKIP` / `mask_STOP` — updateMasks 跳过/停止时各书签 mask 快照
- `mask_FRAME` — 每帧所有书签 mask(数据量大,慎用)

## 解析日志

```bash
python3 -c "
import json
data = json.load(open('ms-log.json'))
from collections import Counter
print(Counter(d['tag'] for d in data))
# 或追踪特定书签的 mask 演变
for d in data:
    if d['tag'] == 'mask_STOP':
        print(json.dumps(d['data'], indent=2, ensure_ascii=False))
"
```

## 排查 mask 残留类问题的思路

mask 残留(书签离开选框后遮罩不更新)通常涉及:
1. `_hovered` 标记是否卡住(拖拽中 mouseleave 被拦截)
2. `updateMasks` 是否跳过该书签(看 mask_SKIP)
3. `blkMask` 算出的值是否正确(看 mask_STOP 的 bx/selL/selR 几何)

抓一次操作日志,看上述 tag 时序,能快速定位是"标记卡住"还是"几何算错"。
